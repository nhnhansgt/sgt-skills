# Sections

This file defines all sections, their ordering, impact levels, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. Performance (perf)

**Impact:** CRITICAL
**Description:** Performance optimizations for database queries, caching, and eager loading. Critical for application scalability and response times.

## 2. Security (security)

**Impact:** CRITICAL
**Description:** Authentication, authorization, and security best practices to protect your application from vulnerabilities.

## 3. Architecture (arch)

**Impact:** HIGH
**Description:** MVC + Service Pattern architecture for maintainable, scalable Laravel applications.

## 4. Coding Standards (std)

**Impact:** HIGH
**Description:** PSR standards, naming conventions, and coding style for consistent, readable code.

## 5. Data Layer (data)

**Impact:** HIGH
**Description:** Eloquent models, relationships, API resources, and pagination best practices.

## 6. Request Handling (request)

**Impact:** MEDIUM-HIGH
**Description:** Form requests, validation, middleware, and routing conventions.

## 7. Views (views)

**Impact:** MEDIUM
**Description:** Blade templates, components, and view organization.

## 8. Events & Testing (events)

**Impact:** MEDIUM
**Description:** Event listeners, observers, and testing strategies.
