# Rule Categories

## Priority 1: Performance (CRITICAL)

Rules that have critical performance impact on Java applications.

## Priority 2: Security (CRITICAL)

Rules that prevent security vulnerabilities and ensure safe data handling.

## Priority 3: Code Quality (HIGH)

Rules for writing clean, maintainable, and readable code following best practices.

## Priority 4: Concurrency (HIGH)

Rules for proper multi-threading and concurrent programming in Java.

## Priority 5: Collections & Streams (HIGH)

Rules for efficient use of Java collections framework and Stream API.

## Priority 6: Exception Handling (HIGH)

Rules for proper exception handling and error management.

## Priority 7: Design Patterns (MEDIUM-HIGH)

Rules for applying common design patterns appropriately.

## Priority 8: Testing (MEDIUM)

Rules for writing effective unit and integration tests.
